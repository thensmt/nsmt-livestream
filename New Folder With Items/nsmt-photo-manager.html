Full Setup Guide — OBS Player Overlay
1
Set Up Firebase (one time, ~20 minutes)
Go to console.firebase.google.com → Create a new project → Name it "nsmt-stats".

Enable Firestore Database (start in test mode).
Enable Authentication → Google sign-in provider.
Go to Project Settings → Your Apps → Add Web App → copy the firebaseConfig object.

Paste that config into the Overlay Config tab above and also into your NSMT Stats app file.
2
Update Your NSMT Stats App to Write Overlay State
Your stat app needs to write a special overlay/active document to Firestore every time a stat is logged. This doc contains the current score, period, and last event. Add this to your saveGame() function:

// In saveGame(), after saving locally:
if (window._firebase && G.status === 'live') {
  const { db, doc, setDoc } = window._firebase;
  const aScore = calcTeamScore(G, 'away');
  const hScore = calcTeamScore(G, 'home');
  const lastEvt = G.events[G.events.length - 1];
  const p = lastEvt ? G[lastEvt.team]?.players?.find(x => x.id === lastEvt.pid) : null;
  const pStats = lastEvt ? G.stats[lastEvt.pid] : null;
  setDoc(doc(db, 'overlay', 'active'), {
    awayName: G.away.name, awayCode: G.away.code, awayScore: aScore,
    homeName: G.home.name, homeCode: G.home.code, homeScore: hScore,
    period: G.currentPeriod, eventName: G.event || 'LIVE',
    lastEvent: lastEvt ? {
      ...lastEvt,
      playerName: p?.name || '',
      playerNum: p?.num || '',
      ptsGame: pStats ? calcPts(pStats) : 0,
    } : null,
    updatedAt: Date.now()
  });
}
3
Set Up Player Headshots from Google Drive
Upload your player headshots to a Google Drive folder.

For each photo: Right-click → Share → Change to "Anyone with the link" → Copy link.

The file ID is the long string in the URL: drive.google.com/file/d/THIS_IS_THE_ID/view

Go to the Player Photos tab on this page, load your roster, and paste each player's Drive file ID into their card. Hit Save.

The overlay will pull the thumbnail automatically using: drive.google.com/thumbnail?id=FILE_ID&sz=w400
4
Add the Overlay to OBS
In OBS, go to your game scene → click + in Sources → add Browser Source.

Settings:
• URL: file:///path/to/nsmt-obs-overlay.html (or host it at overlay.thensmt.com)
• Width: 1920
• Height: 1080
• ✅ Check "Transparent background"
• FPS: 60

Place this Browser Source above your game camera/stream capture in the source list so it appears on top.
5
Test Before Game Day
Open nsmt-obs-overlay.html directly in Chrome to test it. You'll see demo buttons bottom-right.

Click "Demo 2PT" and "Demo 3PT" to see the score pop card fire. Adjust the timing in SETTINGS.displayDuration if you want it to show longer or shorter.

Once Firebase is connected and the stats app is running a test game, the real overlay will fire automatically whenever you log a basket.
6
Game Day Workflow
1. OBS computer opens the overlay URL in its Browser Source (auto-connects to Firebase)
2. Stat-keeper opens NSMT Stats app on their iPad/phone, signs in with Google
3. Start a new game in the stats app
4. Every time a basket is logged → Firebase fires → overlay pops within 1–2 seconds
5. Scoreboard bar updates live at the top of the stream

The stat-keeper and OBS computer don't need to be on the same Wi-Fi. Any internet connection on both sides is all that's needed.
